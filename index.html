06 — TROUBLESHOOTING
트러블 슈팅
문제원인해결
포인트 이중 차감 버그
DB 저장보다 상태 업데이트가 먼저 실행되어 변경된 값을 기준으로 차감이 한 번 더 발생
포인트 차감 → PointHistory 저장 → UserPoint 업데이트 순서로 실행 흐름 재정렬
유사도 점수 균일 현상
오디오 특성 벡터 유사도가 모두 75%로 동일하게 출력되어 변별력 없음
Min-Max 정규화로 오디오 점수를 0.5~1.0 범위로 재분포하여 변별력 확보
정렬 변경 시 재생 오류
정렬 순서 변경 시 PlayerContext 큐가 이전 순서 그대로 남아 다음 곡이 잘못 재생
onChange 핸들러에서 새 정렬 기준으로 큐를 재계산하고 현재 곡 인덱스를 찾아 play() 재호출
새로고침 시 로그인 풀림
Redux는 메모리 상태라 페이지 새로고침 시 초기값으로 리셋됨
Redux-Persist 도입으로 localStorage에 인증 상태 영속화하여 새로고침 후에도 로그인 유지
07 — RETROSPECTIVE
배운 점
이주용
팀장 · AI 추천 / 챗봇
벡터 유사도는 원점수를 그대로 쓰면 안 된다는 것을 깨달았습니다. 코사인 유사도는 특정 범위에 값이 몰릴 수 있어 후처리 없이는 변별력을 얻기 어렵습니다. 프롬프트 엔지니어링으로 hallucination을 줄이고, 외부 API로 사실 검증하는 별도 단계가 필요함을 알게 됐습니다.
김형준
팀원 · 회원 / 굿즈 / 관리자
인증부터 결제까지 풀스택으로 담당하며 데이터 흐름 전체를 설계하고 책임지는 경험을 했습니다. JWT와 Redux 상태 관리를 연결하는 과정에서 서버와 클라이언트가 독립적으로 동작한다는 점을 체감했고, 양쪽 모두에서 보안을 고려해야 한다는 것을 깨달았습니다.
윤기선
팀원 · 게시판 / 랭킹 / 게임
시간 관리, 팀 협업, 타인의 코드를 이해하는 것, 쿼리 이해도 등 기본 역량을 익히는 시간이었습니다. 만들어야 할 것과 실질적으로 언제까지 만들 수 있는지를 현실적으로 조율하는 법을 배웠습니다.
김경민
팀원 · 홈 / 플레이리스트 / 월드컵
프론트엔드부터 백엔드, 외부 API까지 풀스택으로 진행하며 여러 레이어가 맞물리는 구조를 체감했습니다. 코드 한 줄이 팀원의 작업에 영향을 미친다는 책임감이 생겼고, 동작하는 코드를 넘어 구조를 고민하는 습관을 길렀습니다.
